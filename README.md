# 3-Desafio-de-criacao
Criar prompt da newsletter de divulgação interna de novos produtos

🎯 Desafio Criativo: Construindo um Prompt
Bem-vindo ao Desafio Criativo.
Nesta atividade, você seguirá algumas etapas simples que, ao final, irão ajudar a construir um prompt completo e bem estruturado.

💡 Sobre o Exercício
Um bom prompt nasce de intenção clara, contexto e instruções específicas. Durante o desafio, você irá organizar suas ideias e transformá-las em comandos claros.

🧩 Construção Passo a Passo
Cada etapa adicionará uma nova peça ao seu prompt final. Ao concluir todas, você terá um prompt capaz de orientar uma IA a produzir exatamente o que você imaginou.

🚀 Como Resolver
Siga cada passo com atenção, registre suas respostas e, ao final, una todos os elementos para formar seu prompt final.

Aproveite o processo criativo!

Etapa 1 — 🧨 “Postaram no Slack e chamaram de comunicação” (o novo incêndio)
Segunda-feira, 09:07.

Você abre o Slack e dá de cara com três mensagens no #geral:

“🚀 Lançamos o feature X!!” (sem link)
“Atualização no produto Y” (um print cortado)
“Tem mudança no fluxo do Z” (com 18 linhas técnicas e 4 siglas)
Em menos de 10 minutos, o caos começa:

o time de CS pergunta: “isso impacta cliente?”
o time Comercial pergunta: “posso falar disso?”
o time de Produto responde: “tá em rollout, não é pra todo mundo”
e alguém solta: “uai, mas já tá valendo ou não?”
A liderança aparece com uma ordem curta e direta:

“Chega de improviso.
Quero um PROMPT que gere a newsletter interna toda semana, no mesmo padrão, com clareza, CTA e links.
E tem que funcionar mesmo se quem preencher não for expert em escrita.”

Seu objetivo agora: criar um prompt definitivo (template) que transforma informações básicas de lançamentos em uma newsletter interna pronta para publicar.

🎯 Objetivo da etapa
Definir o resultado final do prompt respondendo (em bullets mesmo):

📣 Onde a newsletter vai ser publicada?
(email, intranet, Slack/Teams, ou mais de um)

⏱️ Quanto tempo máximo de leitura?
(ex.: 1 min / 3 min / 5 min)

🧱 Quais seções são obrigatórias na saída?
(ex.: 👋 abertura, 🚀 novidades, 🧠 resumo, 📣 CTA, ❓FAQ)

🗣️ Tom de voz desejado
(3 adjetivos, ex.: claro, direto, humano)

Etapa 2 — 🧾 Briefing simples (o mínimo que garante uma newsletter boa)
Você percebe uma coisa: quanto mais campos, maior a chance de ninguém preencher. Então você troca o “formulário perfeito” pelo formulário possível — o mais enxuto que ainda evita newsletter vazia.

A regra é: se o time preencher isso aqui, a IA consegue fazer o resto com qualidade.

✅ Bloco de entrada (copiar e colar)
📌 Dados da edição

SEMANA/DATA:
TEMA (1 linha):
ONDE VAI PUBLICAR: (email / intranet / Slack/Teams)
🚀 Novidades (repita por item)

Nome do produto/feature:
O que é (1 frase):
Por que importa (1 frase):
Pra quem é:
Como acessar/usar (passo ou link):
Status: (lançado / rollout / beta / piloto)
Links úteis: (docs/demo)
📣 CTA interno: (comece com verbo)
Dúvidas com: (pessoa ou canal)
🧠 Regras rápidas
Se não souber algo: N/A (não inventar).
Sigla? Escreva por extenso na primeira vez.
CTA sempre com verbo: Acesse / Teste / Responda / Envie / Sugira.
🎯 Objetivo da etapa
Ter um briefing curto e preenchível que garanta:

contexto (o que é + por que importa)
ação (CTA)
acesso (como usar + links)
suporte (dúvidas com)

Etapa 3 — 🧠 O prompt “oficial” (template pronto pra rodar toda semana)
Agora que o briefing ficou simples, você só precisa de uma coisa: um prompt que nunca falha. Um texto que, quando alguém colar + preencher, devolve uma newsletter no padrão certo — mesmo se a pessoa estiver com pressa.

Você decide escrever o prompt como se fosse um “contrato”: papel + regras + formato obrigatório + bloco de entrada.

🧩 Prompt Template — Gerador de Newsletter Interna (Novos Produtos)
Copie e cole este prompt. Depois, só preencha o bloco “📌 DADOS DE ENTRADA”.


Você é um(a) especialista em Comunicação Interna de produtos digitais.
Seu trabalho é transformar informações de lançamentos em uma newsletter interna clara, útil e acionável.

🎯 Objetivo
Gerar uma newsletter interna pronta para publicação, com tom interno e estrutura padronizada, a partir dos dados fornecidos.

🧠 Regras
- Escreva para público interno, com tom: claro, direto e humano (sem “marketingzão”).
- Use linguagem simples, evite jargões; se usar siglas, explique na primeira vez.
- Não invente nada: se faltar informação, escreva exatamente “⚠️ Informação pendente”.
- Deixe escaneável: títulos curtos + bullets.
- Sempre inclua: “como acessar/usar”, “CTA interno” e “dúvidas com”.

# 📄 Entrega do Prompt Final
Após concluir o Desafio Criativo, você deverá entregar seu prompt final por meio de um documento no Google Docs.

✍️ Passo 1: Criar o documento
Acesse: https://docs.google.com
Clique em "Em branco" para criar um novo documento.
Dê um título ao documento, por exemplo:
Desafio Criativo – Prompt Final
Cole no documento o prompt que você construiu durante a atividade.
🔗 Passo 2: Tornar o documento público
Para que possamos acessar seu trabalho, é necessário compartilhar o documento de forma pública.

Clique no botão "Compartilhar" no canto superior direito.
Na seção "Acesso geral", clique no menu que normalmente aparece como "Restrito".
Selecione a opção "Qualquer pessoa com o link".
Defina a permissão como "Leitor".
📤 Passo 3: Enviar o link
Clique em "Copiar link".
Cole esse link no local indicado para envio da atividade.
✅ Importante
Antes de enviar, verifique se:

O documento contém seu prompt final completo
O link está configurado como "Qualquer pessoa com o link pode visualizar"
Assim garantimos que seu trabalho possa ser acessado corretamente. 🚀
